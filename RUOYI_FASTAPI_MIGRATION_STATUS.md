# 若依 FastAPI + Next 迁移进度

本文档记录当前若依前后端分离项目迁移到 FastAPI、PostgreSQL、Next.js 的完成情况和后续待办。

## 项目位置

- 原若依项目：`RuoYi-Vue-master`
- Python 后端：`ruoyi-fastapi-backend`
- Next 前端：`ruoyi-next-admin`

## 当前运行信息

- 前端地址：`http://127.0.0.1:3000`
- 后端地址：`http://127.0.0.1:8000`
- 默认账号：`admin / admin123`
- PostgreSQL 容器：`pgsql`
- PostgreSQL 端口：`5433 -> 5432`
- 数据库：`ruoyi_fastapi`
- Redis 容器：`myredis`
- Redis 端口：`9379 -> 6379`

## 已完成

### 后端基础能力

- FastAPI 项目骨架。
- PostgreSQL 数据库连接。
- Redis 会话连接。
- SQLAlchemy 异步模型。
- 若依核心 `sys_*` 表结构迁移。
- PostgreSQL 初始化 SQL：`ruoyi-fastapi-backend/sql/init_postgres.sql`。
- 默认管理员账号初始化。

### 登录与权限

- 登录接口：`POST /login`。
- 退出接口：`POST /logout`。
- 用户信息接口：`GET /getInfo`。
- 动态路由接口：`GET /getRouters`。
- JWT token 登录。
- Redis token 会话。
- 权限字符校验，例如 `system:user:list`。
- 超级管理员 `*:*:*` 权限。

### 系统管理接口

- 用户管理基础 CRUD。
- 角色管理基础 CRUD。
- 菜单管理基础 CRUD。
- 部门管理基础 CRUD。
- 岗位管理基础 CRUD。
- 字典类型基础 CRUD。
- 字典数据基础接口。
- 参数配置基础 CRUD。
- 用户绑定角色、岗位。
- 角色绑定菜单权限。
- 角色数据权限绑定部门。

### 监控日志接口

- 操作日志列表、删除。
- 登录日志列表、删除。

### 前端基础能力

- Next.js 管理端项目。
- 登录页。
- token 本地保存。
- 根据 `/getInfo` 获取用户、角色、权限。
- 根据 `/getRouters` 生成左侧菜单。
- 根据权限字符控制按钮显示。
- 用户、角色、菜单、部门、岗位、字典、参数、日志列表页面（含分页、搜索、CRUD）。
- 通用新增、编辑、删除弹窗。
- 用户表单支持选择角色、岗位。
- 角色表单支持选择菜单权限、数据权限部门。
- **Dict 系统**：字典缓存 + DictTag 彩色标签组件 + selectDictLabel 工具。
- **全局路由守卫**：SidebarLayout 统一鉴权，无 session 自动重定向。
- **个人中心**：个人信息查看/修改、头像上传、修改密码。
- **404 页面**：使用原若依 404 图片。
- **SvgIcon 组件**：使用原若依 88 个 SVG 图标。
- **Modal 弹窗**：modalConfirm/modalAlert 替代 window.confirm。
- **Pagination 分页组件**：复用分页组件。
- **Clipboard**：useClipboard() 剪贴板 hook。
- **Fullscreen 全屏切换**：顶部栏全屏按钮。
- **FileUpload/ImageUpload**：文件/图片上传组件（拖拽上传、预览）。
- **ThemePicker 主题色**：8 种预设主题色切换。
- 中文文案已恢复，源码使用安全写法避免编码损坏。

- 菜单图标使用 lucide-react 非原若依 SVG — 已替换。
- 已删除无用 catch-all 页面 [...slug]。
- 移除各页面冗余的"共 N 条"表格头部文字。

## 暂不建议迁移

这些模块和 Java 技术栈绑定较深，第一期不建议做：

- 代码生成。
- Quartz 定时任务。
- Druid 监控。
- MyBatis 专属实现。
- 若依 Java 工具链相关能力。

## 当前完成度判断

当前版本可认为是 **"若依 RBAC 核心体验完整可运行"**：

- 后端核心权限链路已具备，操作日志/登录日志自动记录。
- 前端可以登录、看菜单、做完整 CRUD（分页/搜索/排序/批量删除/导出）。
- 用户、角色、菜单、部门、岗位、字典、参数的核心表已接入，前端表单完整。
- Dict 缓存系统 + DictTag 组件已实现。
- 个人中心（信息修改/头像上传/密码修改）已实现。
- 404 页面、全屏切换、主题色切换已实现。
- 若依原生 SVG 图标已全部迁移。

但还不是"100% 若依后台体验"：

- 用户导入/导出、角色分配用户、菜单半选逻辑、部门树筛选用户等细节待补。
- 通知公告、在线用户、验证码、服务/缓存监控等模块待做。
- 还需要做完整浏览器功能验收和异常场景测试。
