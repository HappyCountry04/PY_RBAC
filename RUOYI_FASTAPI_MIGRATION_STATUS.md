# 若依 FastAPI + Next 迁移进度

本文档记录当前若依前后端分离项目迁移到 FastAPI、PostgreSQL、Next.js 的完成情况和后续待办。

## 项目位置

- 原若依项目：`RuoYi-Vue-master`
- Python 后端：`ruoyi-fastapi-backend`
- Next 前端：`ruoyi-next-admin`

## 当前运行信息

- 前端地址：`http://127.0.0.1:3000`
- 后端地址：`http://127.0.0.1:8000`
- 默认账号：`admin / admin123`
- PostgreSQL 容器：`pgsql`
- PostgreSQL 端口：`5433 -> 5432`
- 数据库：`ruoyi_fastapi`
- Redis 容器：`myredis`
- Redis 端口：`9379 -> 6379`

## 已完成

### 后端基础能力

- FastAPI 项目骨架。
- PostgreSQL 数据库连接。
- Redis 会话连接。
- SQLAlchemy 异步模型。

- PostgreSQL 初始化 SQL：`ruoyi-fastapi-backend/sql/init_postgres.sql`。
- 默认管理员账号初始化。

- 退出接口：`POST /logout`。

- Redis token 会话。

## 已修复的问题

- 前端 `page.tsx` 编码损坏导致页面一直显示“正在加载后台”。
- 前端 `.next` 缓存损坏导致构建异常。
- 前端 JSX 文本中 `\uXXXX` 原样显示的问题。
- 用户删除接口中未定义变量导致 500。
- 用户编辑未提交事务导致保存不生效。
- 用户状态切换接口未写入状态。
- 编辑用户/角色时误提交空授权数组，导致授权关系被清空。
- 前端请求增加超时和错误提示，避免无限卡住。

## 暂不建议迁移

这些模块和 Java 技术栈绑定较深，第一期不建议做：

- 代码生成。
- Quartz 定时任务。
- Druid 监控。
- MyBatis 专属实现。
- 若依 Java 工具链相关能力。

## 当前完成度判断

当前版本可以认为是“RBAC 后台骨架可运行”：

- 后端核心权限链路已经具备。
- 前端可以登录、看菜单、做基础 CRUD。
- 用户、角色、菜单、部门、岗位、字典、参数的核心表已接入。

但还不是“完整若依后台体验”：

- 页面交互仍简化。
- 树形控件、分页查询、导入导出、日志自动记录、在线用户等能力还需要继续补。
- 还需要做完整浏览器点击验收和异常场景测试。

## 下一步建议

建议按以下顺序继续：

1. 修完善前端基础体验：分页、查询、重置、批量删除。
2. 补用户管理：重置密码、状态切换、部门树、导入导出。
3. 补角色管理：分配用户、树形菜单权限、树形数据权限。
4. 补字典数据维护。
5. 补操作日志自动记录。
6. 补在线用户、缓存监控、服务监控等运维模块。
