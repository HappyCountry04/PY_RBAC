# 若依 Next 管理端

这是若依 FastAPI 后端的 Next.js 管理端，用来验证 Python 版权限后台的核心流程。当前目标不是只做外观相似，而是逐步把若依常用后台页面的操作流程复刻出来。

## 当前已实现

- 登录页，token 本地保存。
- 登录后读取 `/getInfo` 和 `/getRouters`。
- 根据后端动态路由生成左侧菜单。
- 根据权限字符控制按钮显示。
- 用户、角色、菜单、部门、岗位、字典、参数、操作日志、登录日志列表。
- 常用列表分页、查询、重置、刷新。
- 表格工具栏支持新增、单选修改、批量删除、当前页导出。
- 表格工具栏支持搜索区域显示/隐藏、列显隐。
- 用户、角色、岗位、字典、参数支持创建时间范围查询。
- 用户、角色、菜单、部门、岗位、字典、参数的新增、编辑、删除。
- 菜单、部门已支持逗号 ID 批量删除，仍保留“存在子级不能删除”的若依规则。
- 用户管理支持左侧部门树，点击部门过滤用户。
- 用户新增/编辑的归属部门使用树形选择。
- 用户状态切换、重置密码。
- 用户详情弹窗。
- 用户导入 CSV、下载导入模板。
- 用户分配角色独立弹窗，保存到 `/system/user/authRole`。
- 角色状态切换。
- 角色新增/编辑时选择菜单权限。
- 角色数据权限使用独立弹窗维护。
- 角色分配用户、取消授权用户。
- 菜单权限和部门数据权限支持父子联动、半选状态。
- 菜单管理、部门管理按父子关系缩进显示，并支持新增子级。
- 菜单表单支持菜单类型切换时动态显示字段。
- 菜单表单支持图标选择器。
- 字典类型可进入字典数据维护。
- 字典类型和字典数据已按原若依 SQL 对齐：10 个字典类型、29 条字典数据。
- 字典数据支持回显样式字段 `listClass`、样式属性 `cssClass`，列表按标签样式展示。
- 字典页支持刷新缓存兼容接口。
- 参数页支持刷新缓存兼容接口。
- 角色菜单权限、角色数据权限树增加全选/清空操作。
- 后端保存时校验唯一性：用户账号、角色名称、角色权限字符、同级菜单名称、岗位编码、岗位名称、参数键名、字典类型。

## 为什么之前字典数据和原若依不一样

之前为了先跑通 RBAC 和后台基础流程，初始化数据使用的是精简 seed，只放了 `sys_user_sex` 这一个字典类型和 3 条性别数据。

现在已经改为按原若依 `ry_20260417.sql` 初始化：

- `sys_dict_type`：10 条。
- `sys_dict_data`：29 条。
- 当前 Docker PostgreSQL 数据库也已同步。
- `sql/init_postgres.sql` 也已更新，后续重新建库会自动使用完整字典。

## 运行方式

推荐在上级目录使用一键启动脚本：

```powershell
cd E:\my\zhiren\RuoYi-Vue-master
.\start-ruoyi-fastapi-dev.ps1
```

也可以单独启动前端：

```powershell
cd ruoyi-next-admin
npm install
npm run dev -- --hostname 127.0.0.1 --port 3000
```

访问地址：

```text
http://127.0.0.1:3000
```

默认后端地址配置在 `.env.local`：

```text
NEXT_PUBLIC_API_BASE=http://127.0.0.1:8000
```

默认登录账号：

```text
admin / admin123
```
